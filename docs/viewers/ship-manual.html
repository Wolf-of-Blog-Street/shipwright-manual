Observable nervous system

Ship Computer

TARGET / NOT BUILT Design only. No runtime, event store, API, or live feed is claimed.

Role: watch all activity the ship can actually observe, persist it in external append-only state, correlate and redact it, then expose a Bridge API, live feed, history, and operational map. It may transport an authenticated command that already has authority; it never originates, expands, or approves one, writes Brains, moves funds, or lands code.

Observable activity

Events, not private cognition

Model-turn metadata, typed tool calls, orders, queues, reports, boards, workers, workspaces, tests, grants, costs, Brain-write events, landings, and heartbeats.

API, live feed, and history

Commodore + bounded ship views

Target visibility

  • Every seat/session: model, effort, commission hash, health, heartbeat, current envelope.
  • Every order: authenticated issuer, lifecycle, amendment, acceptance, report correlation.
  • Every staff cell: Advisor, PA, Scientist, Engineer queues, wake state, and bounded result.
  • Every build: manager/worker/workspace tree, files, tests, review, candidate, landing.
  • Every resource/memory event: grant/cost/custody and Brain delta/ratification/version.
  • Every state carries truth class: recorded, receipt, reported, derived, configured, or unknown.

Authority-to-acceptance lineage

Command Graph

TARGET / NOT BUILT Current boards, queues, ledgers, and reports are partial ingredients, not an end-to-end graph runtime.

Required lineage: Commodore directive / standing authority -> Captain mission -> objective -> typed route/order -> epic -> task -> attempt -> evidence/report -> explicit acceptance decision.

No orphan work

  • Every executable node has one primary command parent, one root mission, and an authenticated authority source.
  • Secondary contributes-to links never replace primary lineage.
  • Every attempt links to its task, model, grant, and workspace; retries create new attempt IDs.
  • No orphan work is accepted, dispatched, landed, paid, or reported complete.
  • The graph observes lineage. It cannot mint authority or infer approval.

Queue state is not completion

  • queued is not delivered.
  • delivered is not working.
  • reported is not accepted.
  • Complete requires a correlated report, acceptance evidence, and an explicit decision. Only Captain closes ship-level work.

What remains unproven
